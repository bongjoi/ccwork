# 태그 기능 정의서 (확정본)

> `spec-original.md`를 기반으로, 데이터 구조·UI 패턴·엣지 케이스에 대한 인터뷰를 거쳐 확정한 최종 스펙.
> 원본은 보존하고 결정 사항은 이 문서에 누적한다.

## 기능 개요

노트에 태그를 추가하고 관리할 수 있다.

## 기능 요구사항

- 노트에 태그를 추가할 수 있다.
- 노트에 추가된 태그를 삭제할 수 있다.
- 태그 목록을 노트 상세 화면(NoteEditor)에서 확인할 수 있다.

---

## 인터뷰 결과 — 확정 사항

### 1. 데이터 구조

- `Note` 인터페이스에 **`tags: string[]`** 필드를 추가한다.
- `db.json`은 노트 단위 1:N으로 자연스럽게 묶이며, 별도 컬렉션을 두지 않는다.
- 기존 노트(필드 없음) 호환은 컴포넌트에서 `note.tags ?? []`로 안전 접근한다.

### 2. 입력 UI

- 단일 text input + **Enter** 키로 chip(pill) 형태로 추가.
- chip은 노트 제목/내용 아래, 저장 버튼 위 영역에 가로로 나열한다.

### 3. 저장 시점

- chip 추가/삭제는 `NoteEditor`의 **지역 state**에만 머물고, 노트 **"저장" 버튼 클릭 시** title/content와 함께 한 번의 PATCH로 서버에 반영한다.
- 기존 NoteEditor의 controlled component → 저장 버튼 패턴과 동일하게 동작한다.

### 4. 정규화 규칙

- 입력값은 `trim()`만 적용. **대소문자는 보존**한다(사용자 의도 존중).
- 빈 문자열(또는 공백만 입력)은 추가하지 않는다 → **silent return**.
- **IME 조합 중 Enter는 무시한다.** 한글/일본어/중국어 등 IME가 조합 중인 글자를 확정하기 위해 발생시키는 Enter는 태그 추가로 처리하지 않으며, 조합 종료 후 사용자가 명시적으로 누른 Enter만 처리한다. 가드 기준: `e.nativeEvent.isComposing === false`.

### 5. 중복 처리

- trim된 값으로 정확 일치 비교, 중복이면 **silent ignore**.
- alert/에러 메시지를 띄우지 않는다(프로젝트 규칙: alert 금지·silent return).

### 6. 삭제 UX

- 각 chip 우측의 작은 `×` 버튼 클릭 → 즉시 지역 state에서 제거.
- 별도 확인 다이얼로그를 두지 않는다(가벼운 액션).
- 서버 반영은 노트 저장 시점에 이뤄진다.

### 7. 표시 위치

- 노트 상세 화면 = `NoteEditor` 내부에만 표시한다.
- `NoteList`/`NoteItem`(사이드바 미리보기)에는 표시하지 않는다(이번 스코프 단순화).

### 8. 길이/개수 제한

- 별도 제한을 두지 않는다(학습 프로젝트). 빈 문자열 거부만 유지.

### 9. 에러 처리

- 기존 `handleSave`의 `try/catch + console.error` 패턴을 그대로 활용.
- `tags`도 동일 PATCH 페이로드에 실리므로 별도 에러 분기는 두지 않는다.

## 비스코프 (이번 작업 X)

- 태그로 노트 검색/필터링.
- 기존 태그 자동완성/추천(전역 unique 태그 집계).
- 태그 색상·메타데이터(객체 모델).

## 검증 시나리오

1. **추가**: 새 노트에 "react" Enter, "ts" Enter → 저장 → `http://localhost:3001/notes`에서 `tags: ["react","ts"]` 확인.
2. **표시**: 사이드바에서 같은 노트 재선택 → chip 두 개가 동일하게 보임.
3. **삭제**: chip의 `×` → 사라짐 → 저장 → DB에서도 제거됨.
4. **중복**: 이미 있는 태그를 다시 입력 + Enter → 아무 변화 없음(alert 없음).
5. **공백/빈값**: 빈 input 또는 공백만 입력 + Enter → 아무 변화 없음.
6. **기존 노트 호환**: `tags` 필드가 없는 기존 노트를 열어도 빈 chip 리스트로 정상 동작.
7. **한글 IME**: macOS/Windows 한글 IME에서 `테스트` 입력 후 Enter → `테스트` 단일 chip만 생성됨(`["테스트", "트"]` 같은 잔여 chip이 만들어지지 않는다).
