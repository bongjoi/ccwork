2. 상태 흐름 — 새 노트 저장

NoteEditor에서 저장 버튼을 눌렀을 때의 단방향 흐름. 굵은 화살표는 호출, 점선은 응답. 

sequenceDiagram
  autonumber
  actor User
  participant NE as NoteEditor
  participant Ctx as NotesContext
  participant API as api/notes
  participant JS as JSON Server

  User->>NE: 제목/내용 입력 + 저장 클릭
  NE->>NE: setSaving(true)
  NE->>Ctx: addNote(title, content)
  activate Ctx
  Ctx->>API: createNote({title, content})
  activate API
  API->>JS: POST /notes (createdAt/updatedAt 포함)
  JS-->>API: 201 + Note
  API-->>Ctx: Note
  deactivate API
  Ctx->>Ctx: setNotes(prev => [...prev, newNote])
  Ctx-->>NE: Promise resolve
  deactivate Ctx
  NE->>NE: onDone() / setSaving(false)

4. 상태 전이 — App 모드 머신

selectedNoteId × isCreating이 결정하는 세 모드와 사용자 액션에 따른 전이. NoteEditor.handleSave의 saving 플래그는 본 머신과 무관하므로 제외. 

stateDiagram-v2
  [*] --> Idle

  Idle --> Creating: handleNewNote()
  Idle --> Editing: handleSelectNote(id)

  Creating --> Idle: handleDone() (저장 또는 취소)
  Creating --> Editing: handleSelectNote(id)

  Editing --> Creating: handleNewNote()
  Editing --> Editing: handleSelectNote(otherId)
  Editing --> Editing: handleDone()

  note right of Idle
    selectedNoteId = null
    isCreating = false
    안내 화면
  end note

  note right of Creating
    selectedNoteId = null
    isCreating = true
    NoteEditor: 빈 폼
  end note

  note right of Editing
    selectedNoteId = string
    isCreating = false
    NoteEditor: 선택 노트
  end note
